# ResearchBox 后续开发计划（2026-07-11）

## 当前阶段判断

项目已达到“本地单机内测版”：项目、资料导入、笔录校正、AI 编码、定性小结、洞察、报告及多格式导出可以形成完整闭环。尚未达到公网商业上线标准，核心差距集中在服务端基础设施、安全合规、规模性能和自动化验收。

## Phase 1：内测稳定性（P0，1–2 周）

1. 将页面改为路由级懒加载，拆分当前约 1 MB 主包；ExcelJS/XLSX 仅在小结页加载。
2. 增加 AI 长任务队列：进度、取消、失败重试、分块分析和断点续作。
3. 增加 Excel 模板异常校验：受访者列不足、重复维度、合并区域、受保护工作簿。
4. 增加小结生成版本、人工编辑、重新生成单个单元格和证据跳转。
5. 增加 Playwright 端到端测试：导入笔录 → 校正 → 编码 → 小结 → 模板导出。
6. 处理 npm 依赖安全告警，评估并替换存在历史漏洞的间接依赖。

验收：20 份访谈、10 个维度的小结可稳定生成；失败可恢复；导出模板结构与样式不变。

## Phase 2：云端与账号（P0，2–4 周）

1. 正式账号认证、会话管理、找回账号和注销。
2. 服务端工作区、成员、角色和项目权限校验。
3. 云数据库、对象存储、附件加密和租户隔离。
4. 使用 Secret Manager 保存AI Key，停止生产环境从 DOCX 读取。
5. 服务端审计日志、数据备份、恢复和迁移。
6. 同步冲突处理和离线编辑策略。

验收：不同工作区数据严格隔离；权限无法通过修改前端绕过；可完成备份恢复演练。

## Phase 3：商业化与合规（P1，2–3 周）

1. 套餐额度、模型 Token/费用统计、用量告警。
2. 订单、支付、退款、发票和对账。
3. 用户协议、隐私政策、AI 数据处理说明和删除机制。
4. 内容安全、接口限流、上传文件扫描和提示词注入防护。
5. 错误监控、性能监控、成本监控和服务状态页。

验收：计费与用量可对账；敏感数据处理可追溯；严重错误可告警和回滚。

## Phase 4：研究能力增强（P1/P2，持续）

1. 跨访谈人群对比、差异显著主题和反例分析。
2. 标签层级、编码本、共现矩阵和研究员一致性检查。
3. 小结模板字段映射编辑器，支持非标准 Excel 模板。
4. 洞察版本对比、证据强度评分和客户审核流程。
5. 项目知识库、历史研究复用和跨项目检索。
6. 更多报告模板和客户品牌模板。

验收：真实研究员完成项目的人工整理时间下降 50% 以上，AI 洞察均可追溯到原文证据。
